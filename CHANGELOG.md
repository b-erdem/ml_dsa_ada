## [0.1.0-pre] — 2026-05-06

### Added

- Initial implementation of ML-DSA (FIPS 204) for Ada/SPARK.
- All three NIST parameter sets via Alire crate config:
  - `ML_DSA_44` (NIST level 2): 1312 / 2560 / 2420 bytes (PK / SK / sig)
  - `ML_DSA_65` (NIST level 3, default): 1952 / 4032 / 3309 bytes
  - `ML_DSA_87` (NIST level 5): 2592 / 4896 / 4627 bytes
- 256-point 8-layer Cooley-Tukey NTT over Z_q[X]/(X^256 + 1) with
  q = 8 380 417 (precomputed Montgomery-form zeta table).
- Power2Round, Decompose (both γ₂ variants, FIPS 204 §7.5 direct
  form), MakeHint, UseHint rounding primitives.
- Rejection sampling for matrix expansion, secret vector sampling,
  challenge polynomial generation, and gamma1 mask sampling.
- Bit-packing routines for t1, t0, η, z, w1; signature hint encoding.
- KeyGen, Sign (bounded rejection loop, `Max_Sign_Retries = 1000`),
  Verify for all three parameter sets, deterministic and hedged
  modes (caller-supplied 32-byte rnd).
- `ML_DSA.Wipe` zeroisation module called on every secret-touching
  local from `KeyGen` and `Sign`.

### Verified

- **100% SPARK level 1 proof, all three parameter sets**: 1880 (44),
  1882 (65), 1881 (87) verification conditions discharged with z3,
  **zero `pragma Assume`**, zero justified checks. Includes type
  safety, range, overflow, division-by-zero, index, initialisation,
  `Always_Terminates`, and chained pre/post-conditions across the
  rejection loop and the matrix / NTT pipeline.
- **NIST ACVP cross-validation**: bit-exact pk/sk/sig output match
  for all three parameter sets against
  [usnistgov/ACVP-Server](https://github.com/usnistgov/ACVP-Server)
  KeyGen + sigGen test vectors.
- **Empirical constant-time** (Welch t-test): Sign with random
  messages and two distinct keys reports |t| < 1 over 200
  iterations (PASS at dudect threshold 4.5).
- **Empirical constant-time** (cachegrind): last-level data cache
  miss delta between two distinct keys = 0.005 % (1 miss out of
  20,015) over 200 ML-DSA-65 Sign iterations. The 9 % L1-data
  delta reflects FIPS 204 §3.4 rejection-loop iteration-count
  variance, not a secret-leaking access pattern. See
  [CT_AUDIT.md](CT_AUDIT.md).

### Tests

- `test_ml_dsa.adb` — 25 unit tests (field arithmetic, rounding,
  NTT, end-to-end KeyGen → Sign → Verify, tamper detection,
  Sign_With_Self_Verify happy path + pk/sk-mismatch rejection).
- `test_kat_ml_dsa.adb` — self-consistency KAT, 5 hashes per set.
- `test_acvp_full.adb` — full NIST ACVP cross-validation: 25
  KeyGen + 15 SigGen (external/pure/det) + 15 SigVer
  (external/pure) per parameter set, byte-exact.
  Generated by `tests/scripts/gen_acvp_full.py` from the official
  ACVP-Server JSON.
- `ct_harness/bin/ct_ml_dsa_sign` — Welch t-test on Sign timing.
- `ct_harness/docker/run_cachegrind.sh` — D1/LLd cache-miss audit.
- `scripts/ct_disasm_check.sh` — disassembly verification that
  `Reduce.CAddQ`, `Rounding.Decompose`, `Rounding.Make_Hint`, and
  `Poly.Poly_ChkNorm` compile to branchless code (jumps only to
  Constraint_Error helpers and loop-back).

### Constant-time hardening

- `Reduce.CAddQ` — branchless via `Shift_Right_Arithmetic` of the
  bit-reinterpreted I32, ANDed with Q.
- `Rounding.Decompose` — branchless centring and wraparound via
  the same sign-mask trick on the differences `Gamma2 - R0` and
  `Boundary - 1 - R1`. The wraparound branch uses
  `Interfaces.Unsigned_32` modular subtraction directly (rather
  than `I64 → mod 2^32 → U32`) so SPARK at level=1 z3 proves it
  uniformly across all three parameter sets.
- `Rounding.Make_Hint` — unconditional Boolean operators (no
  short-circuit) so every disjunct is evaluated.
- `Poly.Poly_ChkNorm` — uses Ada's `abs` operator, which GNAT 14+
  at -O2 emits as the standard branchless `cltd; xor; sub` idiom.
- `Sign.Verify` — challenge-hash equality via accumulating
  XOR-OR loop.

All four CT-critical functions verified branchless via
`scripts/ct_disasm_check.sh` on x86-64.

### Functional correctness

- `Reduce.Reduce32`'s post-condition states `Result ≡ A (mod Q)`
  via `(I64 (A) - I64 (Result)) mod I64 (Q) = 0`, proven by SPARK
  level-1 z3.
- `Reduce.CAddQ`'s post states `Result = A or Result = A + Q`,
  proven.
- `Rounding.Power2Round`'s existing post `A1 * 2^D + A0 = A` is a
  full functional contract, proven.

### Fault-injection countermeasure

- New procedure `Sign_With_Self_Verify (Sig, Ok, M, Ctx, Rnd, SK,
  PK)`: produces a signature and immediately re-verifies it. If
  verification fails (a transient fault corrupted the signing
  computation, or pk/sk are mismatched), `Sig` is zeroed and `Ok`
  is set to `False`. ~1.5x cost vs plain `Sign`.

See [PROOF_NOTES.md](PROOF_NOTES.md) for the proof techniques and
[CT_AUDIT.md](CT_AUDIT.md) for the constant-time audit.
