# ml_dsa_ada

[![CI](https://github.com/b-erdem/ml_dsa_ada/actions/workflows/ci.yml/badge.svg)](https://github.com/b-erdem/ml_dsa_ada/actions/workflows/ci.yml) [![License: GPL-3.0-only / commercial](https://img.shields.io/badge/license-GPL--3.0--only%20%7C%20commercial-blue.svg)](COMMERCIAL-LICENSE.md) [![SPARK: 1898/1898 proved](https://img.shields.io/badge/SPARK%20proof-1898%2F1898%20VCs%2C%200%20assume-brightgreen.svg)](SECURITY.md)

An ML-DSA ([FIPS 204](https://nvlpubs.nist.gov/nistpubs/FIPS/NIST.FIPS.204.pdf))
implementation for Ada 2022, formally verified with
[SPARK](https://www.adacore.com/about-spark).

The post-quantum digital signature scheme — KeyGen, Sign (bounded
rejection loop), and Verify — is **100% SPARK-proved at level 1
across all three parameter sets** (ML-DSA-44/65/87): ~1881
verification conditions per build, 0 unproved, 0 `pragma Assume`. The
8-layer Cooley-Tukey NTT (forward and inverse), Montgomery reduction,
Power2Round / Decompose / MakeHint / UseHint rounding, rejection
sampling, bit packing, and the rejection-sampling sign loop are all
verified end-to-end.

To our knowledge this is the first SPARK implementation of FIPS 204
and the first ML-DSA implementation in any language with this proof
coverage. The closest formally-verified prior work is libcrux-ml-dsa
(Rust → F\* via hax) which verifies field arithmetic, NTT, and
serialization but not the full sign/verify pipeline. See
[PROOF_NOTES.md](PROOF_NOTES.md) for the proof techniques used.

## Key properties

- **Three parameter sets** — ML-DSA-44, ML-DSA-65 (default), ML-DSA-87
  - Sizes match FIPS 204 §4 (Table 1) byte-for-byte.

  | Set | NIST level | PK | SK | Sig |
  |---|---|---|---|---|
  | ML-DSA-44 | 2 | 1312 | 2560 | 2420 |
  | ML-DSA-65 | 3 | 1952 | 4032 | 3309 |
  | ML-DSA-87 | 5 | 2592 | 4896 | 4627 |

- **Formally verified at SPARK level 2** — every module (Reduce,
  Rounding, Poly, NTT, PolyVec, Symmetric, Sampling, Packing, Wipe,
  Sign) proves cleanly (z3 + cvc5 + Alt-Ergo). 1898 VCs discharged,
  0 unproved, including the machine-proved `Decompose` reconstruction
  congruence `a1 * (2*Gamma2) + a0 = a (mod Q)` alongside
  `Power2Round`'s exact identity.
- **Zero `pragma Assume`** — type-level bounds (`Zeta_Type` subtype),
  six-piece segment invariants on the NTT/InvNTT inner loops, and
  the FIPS 204 §7.5 direct-form `Decompose` replace every escape
  hatch the initial implementation used.
- **Functional correctness contracts** on `Reduce32`
  (`Result ≡ A mod Q`), `CAddQ` (`Result = A or A + Q`), and
  `Power2Round` (`A1 * 2^D + A0 = A`).
- **NIST ACVP cross-validated** — bit-exact KeyGen, SigGen, and
  SigVer output across **165 official ACVP vectors** (55 per
  parameter set, all three sets pass), sourced from
  [usnistgov/ACVP-Server](https://github.com/usnistgov/ACVP-Server).
  See [tests/src/test_acvp_full.adb](tests/src/test_acvp_full.adb)
  generated by
  [tests/scripts/gen_acvp_full.py](tests/scripts/gen_acvp_full.py).
- **Constant-time hardened** — `CAddQ`, `Decompose`, `Make_Hint`,
  and `Poly_ChkNorm` are written so GNAT 14+ at -O2 emits branchless
  code (sign-mask arithmetic, unconditional Boolean ops, the `abs`
  / `cltd; xor; sub` idiom). Verified by
  [scripts/ct_disasm_check.sh](scripts/ct_disasm_check.sh) on x86-64,
  by an empirical Welch t-test driver
  (via the workspace `ct_harness` dudect driver),
  and by cachegrind D1/LLd miss audit
  (cachegrind dual-run; methodology documented in CT_AUDIT.md).
- **Fault-injection countermeasure** — `Sign_With_Self_Verify`
  produces a signature and immediately re-verifies it; on
  verification failure the output is zeroed. Mitigates transient
  faults on the signing computation (~1.5x cost vs plain `Sign`).
- **Secret zeroisation** — [`ML_DSA.Wipe`](src/ml_dsa-wipe.ads) clears
  every secret-touching local on `KeyGen` and `Sign` return.
- **Built on [sha3_ada](https://github.com/b-erdem/sha3_ada)** for SHA-3 and SHAKE128/SHAKE256.
- **No heap allocation** — `pragma Pure`, stack-only.
- **All three test programs pass** (25 unit + 5 self-consistency KAT
  + 55 ACVP) for each of the three parameter sets.

## Installation

```toml
# In your alire.toml
[[depends-on]]
ml_dsa_ada = "*"

[[pins]]
ml_dsa_ada = { path = "../ml_dsa_ada" }
```

Selecting a parameter set at build time:

```toml
[configuration]
ml_dsa_ada.parameter_set = "ML_DSA_65"   # or ML_DSA_44, ML_DSA_87
```

## Usage

```ada
with ML_DSA;
with ML_DSA.Sign;

procedure Demo is
   Seed : ML_DSA.Byte_Array_32 := [others => 0];   -- supply 32 random bytes
   Rnd  : ML_DSA.Byte_Array_32 := [others => 0];   -- per-signature randomness
   Msg  : ML_DSA.Byte_Array := ...;                -- message bytes
   Ctx  : ML_DSA.Byte_Array (1 .. 0) := (others => 0);  -- empty context

   PK   : ML_DSA.Byte_Array (0 .. ML_DSA.PK_Bytes - 1);
   SK   : ML_DSA.Byte_Array (0 .. ML_DSA.SK_Bytes - 1);
   Sig  : ML_DSA.Byte_Array (0 .. ML_DSA.Sig_Bytes - 1);
   Ok   : Boolean;
begin
   ML_DSA.Sign.KeyGen (PK, SK, Seed);
   ML_DSA.Sign.Sign   (Sig, Ok, Msg, Ctx, Rnd, SK);
   if Ok and then ML_DSA.Sign.Verify (Sig, Msg, Ctx, PK) then
      --  Signature valid.
   end if;
end Demo;
```

The library has no entropy source built in — the caller must provide
fresh `Seed` and `Rnd` from a CSPRNG.

## Building & testing

```bash
alr build                                       # build the library
cd tests && python3 scripts/gen_acvp_full.py    # regen ACVP vectors for active set
SDK=macos alr build                             # build the tests
./obj/test_ml_dsa                               # 25 unit tests
./obj/test_kat_ml_dsa                           # 5-test self-consistency KAT
./obj/test_acvp_full                            # 55 NIST ACVP test vectors
```

Verifying constant-time:

```bash
bash scripts/ct_disasm_check.sh                 # branchless-codegen audit (x86-64)
```

Run `gnatprove -P ml_dsa_ada.gpr --level=1 --prover=z3 --timeout=20`
for the full type-safety / termination / functional-contract proof
(~3-5 min on Apple Silicon).

To target a different parameter set, edit
`config/ml_dsa_ada_config.ads` (or use the Alire crate config) and
rebuild.
